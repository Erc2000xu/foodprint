# 高德免费版使用政策

> 状态：项目长期技术约束。最后复核：2026-07-24。

## 决策

Foodprint 在中长期阶段**不采购、不接入，也不依赖**高德开放平台的付费服务、流量包、技术服务许可、增值版本或企业专属能力。地图、地点、行政区、商圈、检索和导航跳转的产品设计，必须仅使用当时高德免费版明确可用的能力与免费配额。

这是一条产品边界，不是“暂时不付费、超限后再购买”的预算建议。任何功能在免费配额、账号权限或授权用途之外，必须：

1. 不上线该功能，或先关闭/降级；
2. 或改用另一种经合规审核的免费替代方案；
3. **不得**自动开通流量包、账户余额扣费、付费许可、专业版/旗舰版或商务服务。

## 合规前提

高德现行服务协议规定，个人认证开发者以个人研究学习目的使用基础服务时，才可在一定免费额度内使用；法人/非法人组织、对公众或商业用途等情形可能需要事先购买技术服务许可。项目负责人必须在每次主体、用途或上线范围变化时复核该条款与控制台权限。

因此，若 Foodprint 不再符合高德免费使用的主体/用途/配额条件，本项目的默认处理是**停用相应高德能力或替换提供商**，而不是采购高德服务。不得把“技术上 API 仍可请求”误认为“仍有使用授权”。

权威依据：

- [高德开放平台服务协议](https://lbs.amap.com/pages/terms/)
- [基础服务计费与配额说明](https://lbs.amap.com/pages/base_service_price)
- [服务升级与流量包说明](https://lbs.amap.com/upgrade)

## Foodprint 允许使用的能力

以下能力必须使用免费版可用的 Web 服务 API、JS API 或 URI API；每次接入前以当前控制台和官方文档为准。

| Foodprint 场景 | 高德能力 | 关键输出 | 实施约束 |
| --- | --- | --- | --- |
| 标记餐厅 | 输入提示、关键字/周边/ID POI 搜索 | `poiid`、名称、地址、坐标、城市/区县、分类 | 仅由服务端/Edge Function 调用；防抖、最少 2 字符、限量返回 |
| 餐厅位置层级 | POI 返回值、逆地理编码 | 省/直辖市、市、区县、街道、`adcode`、周边 POI | `city` 与 `district` 保留高德原始字段；展示为“市级 · 区级”，只去除重复前缀，不推断城市 |
| 商圈归属 | 逆地理编码 `extensions=all` 的 `businessAreas`；POI/AOI 搜索 | 商圈名称、中心点、ID/编码；AOI 可带边界 `polyline` | 按某餐厅坐标实时反查；不维护手工“全北京商圈表” |
| 用户按商圈/地铁找店 | 输入提示 + POI/AOI 搜索 | 用户输入匹配的商圈/交通 POI 与坐标 | 以高德实时建议和坐标半径筛选；不声称有全量本地目录 |
| 行政区筛选 | 行政区域查询 | 行政区名称、`adcode`、中心点、下级区划 | 可动态读取；不以项目种子表作为权威数据源 |
| 展示地图 | 高德 JS API、静态地图 API | 底图、缩放、点标记 | 仅使用免费版允许的地图样式与配额；地图不可用时保留列表检索 |
| 跳转导航 | URI API | 唤起高德 App 或 Web 地图 | Foodprint 只传目的地；不上传或保存用户起点 |
| 自建路线结果页（可选） | 基础路径规划 | 驾车/公交/步行/骑行路线 | 仅在免费能力和配额允许时使用；否则继续使用 URI 跳转，不自建导航 |

## 商圈数据的边界

高德可以提供商圈相关的真实地理数据，但普通 POI 搜索并不提供“整座城市一次性、永久全量商圈目录”。官方搜索接口不支持全量数据导出，单次翻页最多 200 条。

Foodprint 的正确模式是：

```text
餐厅的高德 POI / GCJ-02 坐标
        ↓
逆地理编码（businessAreas）
        ↓
该餐厅当前的真实商圈归属

用户输入“王府井”
        ↓
高德输入提示 / POI-AOI 搜索
        ↓
王府井商圈或交通点的坐标（及可用时的 AOI 边界）
        ↓
筛选共同地图中对应范围的餐厅
```

这样既使用高德作为权威来源，也避免建立必然缺失、还会过期的本地商圈字典。Foodprint 可以保存某餐厅的高德 `poiid`、坐标与一次查询获得的展示缓存，以降低调用量；缓存不是新的权威地理库，必须可刷新、可失效，且不得用于批量抓取或再分发高德数据。

## 免费配额防线

高德配额受认证类型、账户权限和平台政策变化影响。当前官方 FAQ 列出的个人认证 Web 服务日配额示例为：输入提示和 POI 搜索各 100 次/日，行政区、地理/逆地理编码、静态地图、基础路径规划等为 5,000 次/日；实际生效额度以当前控制台为准。

必须遵守：

1. `AMAP_WEBSERVICE_KEY` 只存在于服务端或 Supabase Edge Function Secret，绝不发到浏览器或 Git；
2. 搜索请求防抖、最短关键词、去重、限页；不做全量 POI/商圈采集；
3. 行政区结果可按城市做短期缓存；餐厅商圈只在需要展示或刷新时查询；
4. 每月检查高德控制台用量；达到免费额度的 80% 时先收紧节流与缓存，达到 100% 时停用相应实时调用并显示可理解的降级提示；
5. 不配置自动充值、不购买流量包、不升级 Key 权限；
6. 高德接口、配额、条款或用途发生变化时，先做合规评审再继续发布。

## 明确不在范围内

- 付费流量包、账户余额自动扣费、技术服务许可、企业/商务授权；
- 高级服务（例如需要企业授权或商务咨询的能力）、高精度/深度天气、室内店铺级定位；
- 专业版/旗舰版自定义地图、付费云图/空间数据管理能力；
- 从高德批量导出或维护全量城市 POI、商圈、地铁数据集；
- 将高德数据向第三方再分发，或绕开其调用、配额、授权限制。

## 变更检查清单

任何涉及高德的新 PR 必须回答：

1. 它使用的是哪一个官方免费能力？当前控制台是否允许？
2. 是否会增加搜索、逆地理或地图加载次数？如何限流与缓存？
3. 超出免费配额、Key 未配置、用户拒绝定位时如何降级？
4. 是否保存、批量获取或再分发高德数据？若是，停止并重新设计。
5. 用户主体/用途是否仍满足免费使用许可？若不确定，停止接入并由项目负责人复核。
